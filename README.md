# Material Flow — Stage 3 (универсальный локальный учет)

Закрытое PWA-приложение для учета в малом бизнесе: позиции, склад, продажи,
покупки, расходы, производство, поставщики и история действий. Универсальное —
не привязано к одному бизнесу (газоблоки остаются только в демо-данных).

**Этап 3 — рабочий localStorage-MVP с правильной моделью учета.** Без backend,
Supabase, IndexedDB/Dexie, PWA-плагина, биллинга и реальных приглашений.
Архитектура «company-first» (businessId + createdBy на всех записях) готова к
будущей синхронизации Supabase.

## Стек

Vite · React · TypeScript · Tailwind CSS · React Router · lucide-react (иконки).

## Запуск

```bash
npm install
npm run dev      # дев-сервер
npm run build    # сборка (tsc -b && vite build)
npm run lint     # проверка типов
```

Первый запуск показывает только экран создания компании. По умолчанию бизнес
**пустой** — демо-данные можно загрузить отдельной кнопкой.

## Что нового в Stage 3

- **Флаги поведения позиций**: можно продавать / покупать / учитывать на складе /
  использовать в производстве. Формы фильтруют позиции по флагам.
- **Расходы 2.0** в одной вкладке «Расходы» с тремя группами: покупки на склад,
  разовые расходы, постоянные расходы. У каждой свой список, фильтры и подытог.
- **Покупки на склад**: цена по умолчанию из позиции, «Изменить цену» с причиной,
  стоимость доставки, привязка поставщика, «Повторить» покупку, подытоги.
- **Производство со списанием материалов** (consumedMaterials) и обратной
  корректировкой склада при отмене.
- **Движения склада** (StockMovement): покупка/продажа/производство/списание;
  карточки склада показывают последние движения.
- **Поставщики** с WhatsApp (wa.me из номера) и копированием телефона.
- **Отчеты по периодам** + аналитика продаж по времени суток + простые CSS-графики.
- **Причины отмены** с быстрыми пресетами; мягкое удаление везде.
- **Приглашения** — честный «скоро» вместо фейкового кода.
- **Мягкий ambient-фон** (учитывает prefers-reduced-motion).
- **Миграция** старых данных Stage 2 → Stage 3 (storageVersion).

## Структура

```
src/
  app/         App.tsx, routes.tsx (company-gate)
  context/     AppDataContext.tsx — единый источник правды + все действия
  lib/         storage, migrate, items, format, permissions, reports, contacts
  data/        icons.tsx, seedData.ts (демо)
  components/  ui/*, layout/*, business/*
  pages/       Setup, Dashboard, Sales, Expenses, Production, Stock, Items,
               Reports, Suppliers, Activity, Settings, More
  types/       index.ts
  styles/      globals.css
```
