# Material Flow — Stage 2 (локальный MVP)

Учет продаж, расходов, производства и склада для одной компании
(производство и продажа газоблоков, клея, материалов, доставки).

**Этап 2 — рабочий MVP на localStorage.** Данные хранятся на устройстве,
формы реально сохраняют записи, склад/дашборд/отчеты/журнал считаются из
состояния приложения. Архитектура «company-first» (businessId + createdBy на
всех записях), чтобы будущая синхронизация Supabase не требовала переписывания.

Без backend, Supabase, IndexedDB/Dexie, PWA-плагина и реальной авторизации.

## Стек

- Vite + React + TypeScript
- Tailwind CSS
- React Router
- lucide-react (только иконки)

## Запуск

```bash
npm install
npm run dev      # дев-сервер
npm run build    # сборка (tsc -b && vite build)
npm run lint     # проверка типов (tsc --noEmit)
```

Первый запуск показывает только экран создания компании. После создания
открывается основное приложение с нижней навигацией.

## Архитектура данных

- `src/types/index.ts` — все типы (Business, CurrentUser, Membership, Item,
  Sale, Expense, Production, ActivityLog, AppState).
- `src/lib/storage.ts` — единственный модуль, работающий с localStorage
  (ключи `materialFlow.*`), плюс `uid()` / `nowIso()`.
- `src/lib/format.ts` — ₸/числа/даты.
- `src/lib/permissions.ts` — роли owner/admin/member (UI-уровень).
- `src/data/seedData.ts` — демо-данные ТОЛЬКО для первичного засева.
- `src/context/AppDataContext.tsx` — единый источник правды и все действия
  (createCompany, addItem, archiveItem, addSale, cancelSale, addExpense,
  cancelExpense, addProduction, cancelProduction, resetDemoData). Каждое
  действие обновляет React-состояние, localStorage и журнал действий.

Страницы читают данные и вызывают действия только из `AppDataContext` —
прямого доступа к localStorage в страницах нет.

## Что работает

- Создание компании, локальный пользователь, владелец-membership.
- Добавление товаров/материалов/услуг с произвольными атрибутами и шаблонами.
- Продажи (списывают склад), расходы, производство (пополняет склад).
- Soft-delete (отмена с причиной) для продаж/расходов/производства; склад
  восстанавливается/реверсируется. Архив для товаров.
- Дашборд, склад, отчеты, журнал считаются из активных записей в реальном
  времени; отмененные записи исключаются из сумм.
- Журнал действий с фильтрами и событиями отмены.
- Настройки: компания, сотрудники, плейсхолдер приглашения, сброс демо-данных
  (только владелец).
- Данные сохраняются после перезагрузки.
